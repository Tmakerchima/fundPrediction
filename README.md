# FundLens MVP

一个覆盖中国场外公募基金、场内 A 股和美股的趋势研究网页。顶部可以切换三个独立市场页面；基金页使用 5–60 个交易日预测，股票页使用 10 个交易日研究窗口。

## 快速启动

需要 Node.js 20+，不需要安装第三方依赖。

```bash
npm start
```

打开 `http://127.0.0.1:4173`。开发模式使用 `npm run dev`，测试使用 `npm test`。

### 配置 Qwen 证据审查

密钥只从服务端环境变量读取，不能放进前端或提交到仓库。PowerShell 启动示例：

```powershell
$env:DASHSCOPE_API_KEY="替换为已轮换的新密钥"
$env:QWEN_MODEL="qwen-plus"
npm start
```

股票详情和基金详情中可以粘贴财报、公告、季报或新闻原文。Qwen 只返回事实、风险、催化因素以及它与量化信号是否冲突；原始预测和回测值保持只读。当前 MVP 不会自行联网寻找公告，避免把搜索摘要或不明来源误当成正式披露。

## 模型说明

- 数据：东方财富公开基金历史净值，后端缓存 10 分钟。
- 预测：45% 阻尼局部趋势 + 35% 指数加权动量 + 20% 均值回归修正。
- 区间：用最近 60 个交易日对数收益波动生成 80% 与 95% 经验区间。
- 验证：最近样本的走步式样本外验证，展示 MAPE 与方向命中率。
- 基准：展示相对“未来净值等于当前净值”随机游走基准的样本外 R²；负数表示模型不如朴素基准。
- 评分：不再使用主观加减分表；评分 = `NormalCDF(预测对数收益 / 预测期波动) × 100`，解释为模型分布假设下的盈利概率，而不是收益保证。
- 组合：主页提供三套替代组合——低波动现金安全边际代理、Markowitz 风险收益均衡、带现金上限的受限动量；权重采用逆波动率并计入候选基金历史相关性。

## 近两周研究组合

首页组合先从股票、混合、指数、债券和 QDII 五类基金的近一月候选中预筛选，再运行 10 个交易日预测、样本外验证和风险过滤。组合权重不是来自某位投资人的“神奇比例”：它是逆波动率的工程化风险预算。Graham 的安全边际需要估值和基本面数据，本 MVP 只有净值，因此只使用现金仓位和回撤约束作为代理；不能声称已经实现 Buffett/Graham 的完整选股理论。

这个过程仍存在幸存者偏差、动量预筛选偏差和多重检验问题。当前单基金回测没有覆盖“在每个历史时点重新对全市场选出前五名”的完整策略，因此榜单属于研究关注池，不是经过验证的可交易策略。

## 重要限制

基金净值通常仅在交易日收盘后更新，无法像股票一样实时成交。预测只使用历史净值，不包含基金持仓实时变化、基金经理变更、宏观事件、费率、申赎确认日或个人财务情况。页面输出不构成投资建议，不保证未来收益；真实投资前请核对基金合同、最新公告与自身风险承受能力。

## A 股与美股页面

- A 股：`/a-shares.html`，腾讯前复权日线，15 支高流动性代表股研究池。
- 美股：`/us-stocks.html`，Nasdaq 历史日线，15 支高流动性大型股研究池。
- 策略：SMA50/200 趋势过滤、风险调整概率预测、行业去重、交易成本和非重叠样本外验证。
- 机构参考：只采用 State Street、JPMorgan 和 Jane Street 对外公开的多因子、组合风险、流动性与执行原则；其真实生产模型均为专有资产，本项目没有也不声称复制。
- GitHub 工程参考：Qlib 的研究工作流，以及 vectorbt / Backtesting.py 的可复现回测思想。GitHub 热度不能证明策略具有未来超额收益。

股票回测尚未完整模拟滑点、税费、涨跌停、停牌、汇率和实际成交。推荐卡片是研究候选而不是买卖指令；样本外 R² 小于或等于 0 时，模型未战胜随机游走基准，应降低或放弃该信号。
